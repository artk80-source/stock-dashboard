Step 4 — Backend endpoints:
All endpoints return JSON with this shape: {"data": ..., "provider_status": {"finnhub": "ok"|"error", "alpha_vantage": "ok"|"error"|"skipped", "yfinance": "ok"|"error"}}.

GET /api/health — pings all three providers, returns per-provider status, response times, and overall "status": "ok"|"degraded"|"down". Returns HTTP 200 even when degraded (200 = the API itself is reachable; the provider status field tells the truth about underlying providers).
GET /api/catalysts?lookback_hours=2&min_sentiment=0.3&limit=20 — main endpoint. Fetches general market news from Finnhub, filters to positive sentiment within the window, groups by ticker, enriches each ticker with quote (Finnhub) + fundamentals (yfinance). Returns sorted by sentiment descending.
GET /api/stock/{symbol}/news?include_sentiment=true — company news via Finnhub /company-news, optionally enriched with Alpha Vantage sentiment scores. Parse defensively, return {"news": []} on total failure.
GET /api/stock/{symbol}/analysis — combined day-trade + long-term metrics. Day-trade: current price, open, day high/low, previous close, intraday %, volume vs average (Finnhub). Long-term: P/E, market cap, 52w high/low, dividend yield, analyst target, YTD % (yfinance).
GET /api/stock/{symbol}/history?period={period} — periods 1d, 5d, 1mo, 3mo, 6mo, 1y via yfinance.
